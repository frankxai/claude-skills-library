# Reference Deconstruction

## Rule

**Never copy a reference. Extract the system behind it.** A reference tells you *why* something feels premium; you then build an original execution of that principle for this brand. Copying a specific site is both a taste failure and a legal/brand risk.

## Deconstruction axes

For every reference, analyze:

1. First-viewport composition
2. Typography (faces, scale, tracking, rhythm)
3. Grid / layout (and where it breaks)
4. Color / material
5. Hero object / metaphor
6. Motion grammar
7. Scroll behavior
8. Interaction behavior
9. Density rhythm
10. Conversion strategy
11. What makes it feel premium
12. What should **not** be copied (the parts specific to that brand)

## Reference board requirements

A useful board has: **3 typography** references · **3 motion** references · **2 3D/object** references · **2 layout** references · **2 negative** references (what to avoid). Curate intentionally — a board of 30 screenshots is noise.

## Translation format

For each reference, produce:

```
Reference:        <name / url / screenshot>
Observed pattern: <what you literally see>
Underlying principle: <why it works>
Reusable rule:    <the constraint we adopt>
Do NOT copy:      <the brand-specific part we must originate ourselves>
```

## Example

```
Reference:        a best-in-class model-page hero
Observed pattern: one oversized display line, vast quiet, a single slow-drifting object, 2-color palette
Underlying principle: restraint + one focal point = perceived authority
Reusable rule:    hero = 1 display line + 1 sub + 1 CTA + 1 focal object; ≥50% quiet; ≤2 accents
Do NOT copy:      their exact wordmark motion, their specific object, their palette
```

## Calibration set (study, never clone)

The bar is set by interfaces that feel inevitable and expensive: Linear, Vercel, Anthropic/Claude, Apple product pages, Stripe, plus interaction-design references (rauno.me, emilkowalski). Study the *grammar* monthly; ship original execution.
